  <action>
Create backend/app/services/image_linking.py following Phase 3 service patterns (csv_merge.py).

Implement two functions per RESEARCH.md Example 1:

**1. normalize_artikelnummer(art_nr: str) -> str:**
- Strip whitespace: art_nr.strip()
- Convert to lowercase: .lower()
- Return normalized string
- Handles edge case from RESEARCH.md Pitfall 2 (whitespace in artikelnummer)

**2. link_images_to_products(merged_products_path: Path, image_mapping_path: Path) -> ImageLinkResult:**

Algorithm (from RESEARCH.md Pattern 1):
1. Load merged_products.json as list of dicts (json.load)
2. Load .planning/manual_image_mapping.json (json.load)
3. Build case-insensitive lookup index:
   ```python
   image_index = {
       normalize_artikelnummer(art_nr): images
       for art_nr, images in image_mapping["mappings"].items()
   }
   ```
4. For each product in merged_data:
   - Normalize product["artikelnummer"]
   - Lookup in image_index (get normalized_id, default to [])
   - Add matched_images to product["data"]["images"]
   - If matched_images not empty: product["sources"]["images"] = "image_mapping"
   - Else: product["sources"]["images"] = None
   - Count products_with_images
5. Save enhanced merged_products.json (json.dump with ensure_ascii=False, indent=2)
6. Calculate statistics:
   - total_products = len(merged_data)
   - products_without_images = total_products - products_with_images
   - matched_ids = set of normalized artikelnummer from products
   - unused_mappings = count of image_index keys not in matched_ids
7. Return ImageLinkResult with all stats

**Key decisions (from RESEARCH.md):**
- Use Path from pathlib for file operations
- Case-insensitive matching via normalize_artikelnummer (IMAGE-03)
- Empty array for missing images, never omit field (IMAGE-04)
- Preserve all images from manual_image_mapping.json (IMAGE-02)
- Copy image paths verbatim, don't validate file existence (RESEARCH Pitfall 4)

**Imports:**
