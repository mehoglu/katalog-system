<task type="auto">
  <name>Task 2: Text Enhancement Service</name>
  <files>backend/app/services/text_enhancement.py</files>
  <action>
Create `enhance_product_texts()` service function implementing TEXT-01 through TEXT-04:

**Function signature:**
```python
async def enhance_product_texts(
    merged_products_path: Union[str, Path],
    batch_size: int = 30
) -> EnhancementResult
```

**Implementation (per CONTEXT D-01 through D-16):**

1. **Load products:** Read merged_products.json (handle wrapper format from Phase 3)

2. **Batch processing:** Split products into chunks of `batch_size` (default 30)

3. **For each batch:**
   - Build prompt with system message:
     ```
     You are enhancing German product catalog texts. Rules:
     - Improve readability and professionalism
     - Preserve ALL measurements (mm, kg, Stück, etc.)
     - Preserve ALL technical codes (VE, KLS, WS, TL, etc.)
     - Expand abbreviations but keep originals in context
     - Fix capitalization and punctuation
     - Do NOT hallucinate or invent details
     - Return JSON array matching input structure
     ```
   
   - User message contains batch as JSON array:
     ```json
     [
       {"artikelnummer": "...", "bezeichnung1": "...", "bezeichnung2": "..."},
       ...
     ]
     ```
   
   - Call OpenAI GPT-4 (via AsyncOpenAI client from settings)
   - Parse JSON response to get enhanced texts
   - **Quality check:** Verify all measurements/codes from original present in enhanced
   - If quality check fails: log warning, use original text
   
4. **Update products:** Add enhanced texts to `product["data"]`:
   - `bezeichnung1_enhanced`
   - `bezeichnung2_enhanced`
   - Add source tracking: `sources["bezeichnung1_enhanced"] = "llm_enhancement"`

5. **Save:** Write updated merged_products.json (preserve wrapper structure)

6. **Return:** EnhancementResult with statistics

**Error handling:**
- Skip products with missing bezeichnung1
- Continue batch on individual product errors
- Log skipped products

Follow Phase 2 csv_analysis.py pattern for OpenAI integration. Use Phase 4 image_linking.py pattern for file handling.
