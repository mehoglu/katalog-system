<task type="auto">
  <name>Task 2: Implement table rendering and inline editing</name>
  <files>frontend/js/review.js</files>
  <action>
Create review.js with:

1. Page initialization:
   - Get upload_id from localStorage (set during upload) or URL param
   - Fetch products from GET /api/review/{upload_id}
   - Render table rows
   - Show product count

2. Table rendering function:
   - For each product, create table row with cells:
     * Artikelnummer (read-only, bold)
     * Bezeichnung1 (editable contenteditable=true)
     * Bezeichnung1_enhanced (read-only, different style if exists)
     * Bezeichnung2 (editable)
     * Preis (editable)
     * Breite, Höhe, Tiefe (editable dimensions)
     * Gewicht (editable)
     * Bilder (display thumbnail if exists, count if multiple)
     * Sources (show icon indicating source: CSV1, CSV2, LLM-enhanced, manual_edit)

3. Inline editing:
   - Use contenteditable=true for editable cells
   - On blur event, detect if value changed
   - If changed, send PATCH /api/review/{upload_id}/product with:
     - artikelnummer (from row)
     - field_name (from cell data attribute)
     - field_value (cell textContent)
   - Show loading indicator on cell during save
   - Show success (green border flash) or error (red border + alert)
   - Update source indicator to "manual_edit" icon

4. Filter/search:
   - Input box filters table rows by text match (any column)
   - Update product count to show "X of Y Produkte"

5. Error handling:
   - Display error if API calls fail
   - Retry logic for PATCH failures

Follow modern vanilla JS patterns (no jQuery). Use async/await for fetch calls.
Use data attributes for field names: data-field="bezeichnung1"
  </action>
  <verify>
    <automated>grep -q "contenteditable" frontend/js/review.js && grep -q "fetch.*api/review" frontend/js/review.js && echo "✓ JS logic implemented"</automated>
  </verify>
  <done>review.js renders table, enables inline editing, saves on blur, handles errors</done>
