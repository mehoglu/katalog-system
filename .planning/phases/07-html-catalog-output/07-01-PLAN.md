<task type="auto">
  <name>Task 2: Create catalog generation service</name>
  <files>backend/app/services/catalog_generator.py</files>
  <action>
Create catalog generation service with Jinja2 templating.

**Functions:**

1. `generate_catalog(upload_id: str) -> CatalogGenerationResult`:
   - Read merged_products.json from upload directory
   - Create output directory: `uploads/{upload_id}/catalog/`
   - Load Jinja2 templates (product_template.html, index_template.html)
   - For each product:
     * Render product_template with product data
     * Write to `catalog/{artikelnummer}.html`
   - Render index_template with all products
   - Write to `catalog/index.html`
   - Return CatalogGenerationResult (total_products, files_generated, output_path)

2. `format_price(price: str) -> str`:
   - Format price string as EUR (e.g., "89.99" -> "89,99 €")

3. `get_image_path_or_placeholder(bild_paths: list) -> str`:
   - Return first image path if available, else placeholder text

**Error Handling:**
- Raise FileNotFoundError if merged_products.json missing
- Handle missing fields gracefully (show "N/A" for empty values)
- Log generation progress (every 50 products)

**Dependencies:**
- Jinja2 for templating
- Pathlib for file operations
- JSON for reading merged_products.json

Follow async pattern from other services in the codebase.
  </action>
  <verify>
    <automated>docker-compose run --rm backend python3 -c "from app.services.catalog_generator import generate_catalog; print('✓ Service imports successfully')"</automated>
  </verify>
  <done>Catalog generation service created with Jinja2 rendering logic</done>
